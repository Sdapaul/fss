# 금융감독원 비조치의견서·법령해석 자동 알림 시스템 사용법

## 이 시스템이 하는 일

금융감독원 포털(better.fsc.go.kr)에 새로운 **비조치의견서** 또는 **법령해석** 이 등록되면, 자동으로 이메일로 알려주는 시스템입니다.

- 매일 **오전 7시 / 오후 3시** 에 포털을 자동 확인
- 신규 항목이 있으면 이메일 발송 (제목, 원문 URL, AI 요약 포함)
- 이메일에 **엑셀 파일 첨부** (당일 발송 항목 목록)
- 발송 이력을 기록해 중복 발송 방지

---

## 처음 설정하는 방법

### 1단계 — GitHub 저장소 준비

이 코드를 본인의 GitHub 계정에 fork(복사)하거나 직접 저장소를 만들어 올립니다.

> GitHub 계정이 없다면 https://github.com 에서 무료로 만들 수 있습니다.

---

### 2단계 — Gmail 앱 비밀번호 발급

이메일 발송에는 Gmail을 사용합니다. 일반 비밀번호가 아닌 **앱 비밀번호** 를 따로 발급해야 합니다.

1. Gmail 로그인 → 구글 계정 관리 → **보안** 탭
2. 2단계 인증이 **켜져 있어야** 합니다 (꺼져 있으면 먼저 설정)
3. 검색창에 "앱 비밀번호" 입력 → 새 앱 비밀번호 생성
4. 이름은 아무거나 입력 (예: `FSS알림`) → **만들기**
5. 화면에 표시되는 **16자리 비밀번호** 를 복사해 둡니다

---

### 3단계 — Gemini API 키 발급 (AI 요약용, 선택사항)

AI 요약 기능을 쓰려면 Google Gemini API 키가 필요합니다. 없어도 알림은 정상 발송됩니다.

1. https://aistudio.google.com 접속
2. 로그인 후 **Get API Key** 클릭
3. 생성된 API 키 복사

---

### 4단계 — GitHub Secrets 등록

GitHub 저장소에서 **Settings → Secrets and variables → Actions → New repository secret** 을 눌러 아래 4가지를 등록합니다.

| Secret 이름 | 입력할 값 | 예시 |
|---|---|---|
| `FSS_EMAIL_SENDER` | 발신 Gmail 주소 | `myemail@gmail.com` |
| `FSS_EMAIL_PASSWORD` | 2단계에서 발급한 앱 비밀번호 | `abcd efgh ijkl mnop` |
| `FSS_EMAIL_RECIPIENTS` | 수신자 이메일 (여러 명이면 쉼표로 구분) | `a@company.com,b@company.com` |
| `GEMINI_API_KEY` | 3단계에서 발급한 API 키 (없으면 생략 가능) | `AIzaSy...` |

> Secrets는 저장 후 다시 볼 수 없으니 잘못 입력했다면 삭제 후 다시 등록하세요.

---

### 5단계 — GitHub Actions 활성화 확인

저장소의 **Actions** 탭을 클릭해서 워크플로가 활성화되어 있는지 확인합니다.  
비활성화 상태라면 "I understand my workflows, enable them" 버튼을 클릭합니다.

설정이 끝나면 **별도 작업 없이 자동으로 운영**됩니다.

---

## 이메일은 어떻게 오나요?

신규 항목이 있을 때만 이메일이 발송됩니다. 이메일 내용은 다음과 같습니다.

```
제목: [FSS 알림] 비조치의견서·법령해석 신규 3건 (2026.04.23)

─ 법령해석 ──────────────────────────────
● 자본시장과 금융투자업에 관한 법률 제○○조 관련 질의  (2026-04-23)
  🔗 원문: https://better.fsc.go.kr/...
  [AI요약] 질의자는 펀드 운용 중 ... 금융위원회는 ... 라고 회신하였습니다.

─ 비조치의견서 ───────────────────────────
● ...
```

- 제목을 클릭하면 금융감독원 원문 페이지로 바로 이동합니다.
- 중요 법률명은 노란색으로 강조 표시됩니다.
- 엑셀 파일이 첨부되며, 당일 발송된 항목 전체가 담겨 있습니다.

---

## 자동 실행 일정

| 워크플로 | 실행 시각 (한국 시간) | 하는 일 |
|---|---|---|
| 알림 발송 | 매일 오전 7시, 오후 3시 | 포털 확인 후 신규 항목 이메일 발송 |
| 누락 검증 | 매일 오전 8시 | 발송 이력과 포털 항목 비교, 누락 여부 확인 |

---

## 수동으로 즉시 실행하는 방법

### 알림 즉시 발송
1. GitHub 저장소 → **Actions** 탭
2. 왼쪽 목록에서 **FSS 비조치의견서·법령해석 알림** 클릭
3. 오른쪽 **Run workflow** 버튼 클릭 → **Run workflow**

### 기간별 엑셀 추출
발송 이력을 특정 기간으로 엑셀 파일로 내려받을 수 있습니다.

1. GitHub 저장소 → **Actions** 탭
2. 왼쪽 목록에서 **FSS 이력 엑셀 추출** 클릭
3. **Run workflow** → 시작일·종료일 입력 (예: `2026-01-01` ~ `2026-04-23`) → 실행
4. 실행 완료 후 해당 실행 결과를 클릭 → 하단 **Artifacts** 에서 엑셀 파일 다운로드

---

## 수신자 이메일 추가·변경

1. GitHub 저장소 → **Settings → Secrets and variables → Actions**
2. `FSS_EMAIL_RECIPIENTS` 항목 → **Update** 클릭
3. 새 값 입력 (쉼표로 구분): `a@company.com,b@company.com,c@company.com`
4. **Save** 클릭

## 자주 묻는 질문

**Q. 신규 항목이 없으면 이메일이 안 오나요?**  
네. 신규 항목이 없는 날에는 이메일이 발송되지 않습니다.

**Q. GEMINI_API_KEY를 등록하지 않으면 어떻게 되나요?**  
AI 요약 없이 제목, URL, 등록일만 포함된 이메일이 발송됩니다. 나머지 기능은 동일합니다.

**Q. Actions 탭에서 빨간 X(실패) 표시가 나타났어요.**  
해당 실행을 클릭해서 로그를 확인합니다. 대부분의 원인은 다음과 같습니다.
- Gmail 앱 비밀번호가 잘못 입력됨 → Secrets 재등록
- 포털 서버 일시 장애 → 다음 자동 실행 때 자동 복구
- 누락 검증에서 누락 항목 발견 → 로그에서 항목 확인 (알림 발송에는 영향 없음)

**Q. 포털에 있는 과거 항목이 갑자기 이메일로 오지 않나요?**  
오지 않습니다. 시스템은 최근 3일 이내 등록된 항목만 확인하며, 이미 발송된 항목은 재발송하지 않습니다.

**Q. 이메일 발신 주소를 바꾸고 싶어요.**  
`FSS_EMAIL_SENDER`와 `FSS_EMAIL_PASSWORD` Secrets를 새 Gmail 계정으로 업데이트하면 됩니다.

## 파일 구조 (참고용)

```
├── main.py            # 핵심 실행 파일 (포털 확인 → 이메일 발송)
├── scraper.py         # 금융감독원 포털 데이터 수집
├── emailer.py         # 이메일 생성 및 발송
├── summarizer.py      # Gemini AI 요약
├── db.py              # 발송 이력 저장 (SQLite)
├── validator.py       # 누락 항목 검증
├── export_excel.py    # 기간별 엑셀 추출
├── seen_items.json    # 발송 이력 목록 (자동 관리)
└── .github/workflows/
    ├── fss_notify.yml    # 자동 알림 스케줄
    ├── fss_validate.yml  # 누락 검증 스케줄
    └── fss_export.yml    # 엑셀 추출 (수동)
```
